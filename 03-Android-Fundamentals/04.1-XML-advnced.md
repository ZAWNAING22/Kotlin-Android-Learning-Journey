# Android UI Basics with XML (Improved Notes)

Before **Jetpack Compose**, Android UI was built using **XML layouts**.

Even in **2026**, understanding XML is important because:

• Many **legacy apps still use XML**
• You will often **read existing codebases**
• **University exams** still focus on XML layouts
• Some companies maintain **hybrid apps (XML + Compose)**

---

# 1️⃣ What is XML in Android?

**XML (eXtensible Markup Language)** is used to **define the user interface** of an Android app.

It describes:

• UI structure
• Layout hierarchy
• View components

It separates:

| UI Design | Logic  |
| --------- | ------ |
| XML       | Kotlin |

Example:

```xml
<TextView
    android:text="Hello World"/>
```

The **design is written in XML**, while **behavior is written in Kotlin**.

---

# 2️⃣ Where XML Files Are Located

All UI layouts are inside:

```
res/layout/
```

Example files:

```
res/layout/activity_main.xml
res/layout/activity_login.xml
```

Each **Activity or Fragment usually has its own XML layout**.

---

# 3️⃣ Basic Layout Structure

Example layout:

```xml
<?xml version="1.0" encoding="utf-8"?>

<LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World" />

    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Click Me" />

</LinearLayout>
```

Important idea:

**Layouts contain Views**, forming a **View Hierarchy (Tree Structure)**.

Example hierarchy:

```
LinearLayout
 ├── TextView
 └── Button
```

---

# 4️⃣ The Three Most Important Layouts

You don't need to learn every layout. Focus on these **three core layouts**.

---

## 🔹 LinearLayout

Arranges elements **in a row or column**.

Example:

```xml
android:orientation="vertical"
```

Vertical layout:

```
TextView
Button
ImageView
```

Horizontal layout:

```
TextView | Button | ImageView
```

### Important Attribute: `layout_weight`

It distributes space between views.

Example:

```xml
<Button
    android:layout_weight="1"/>
```

This makes views **share available space equally**.

---

## 🔹 ConstraintLayout (Most Important)

This is the **modern XML layout recommended by Google**.

It allows you to **position views relative to other views or the parent**.

Example:

```xml
<Button
    android:id="@+id/button"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"

    app:layout_constraintTop_toTopOf="parent"
    app:layout_constraintStart_toStartOf="parent"/>
```

This means:

• Top aligned to parent
• Start aligned to parent

ConstraintLayout helps:

✔ Avoid deep nested layouts
✔ Build responsive UIs
✔ Improve performance

---

## 🔹 FrameLayout

Used to **stack views on top of each other**.

Example use cases:

• Fragments
• Overlays
• Loading screens

Example structure:

```
FrameLayout
 ├── ImageView
 └── TextView
```

Text appears **on top of image**.

# 6️⃣ Important View Attributes

| Attribute     | Meaning              |
| ------------- | -------------------- |
| layout_width  | Width of view        |
| layout_height | Height of view       |
| match_parent  | Fill available space |
| wrap_content  | Fit content size     |
| id            | Unique identifier    |
| margin        | Space outside view   |
| padding       | Space inside view    |

Example:

```xml
android:layout_margin="16dp"
android:padding="8dp"
```

# 7️⃣ Android Resource System (Important for Exams)

Android uses a **resource system** to organize app assets.

Inside:

```
res/
```

Important folders:

| Folder   | Purpose                 |
| -------- | ----------------------- |
| layout   | UI XML files            |
| drawable | images and shapes       |
| values   | strings, colors, styles |
| mipmap   | launcher icons          |

---

## strings.xml

Avoid hardcoding text.

Example:

```xml
<string name="app_name">MyApp</string>
```

Use in layout:

```xml
android:text="@string/app_name"
```

Benefits:

✔ Translation support
✔ Easy reuse

---

## colors.xml

Define color palette.

Example:

```xml
<color name="primary">#6200EE</color>
```

Use in UI:

```xml
android:textColor="@color/primary"
```

---

# 8️⃣ Connecting XML to Kotlin

XML defines the UI, but Kotlin handles **behavior**.

In Activity:

```kotlin
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)

    setContentView(R.layout.activity_main)
}
```

This **loads the XML layout**.

---

# 9️⃣ Accessing Views (findViewById)

Old method:

```kotlin
val button = findViewById<Button>(R.id.button)
```

Add click action:

```kotlin
button.setOnClickListener {
    Toast.makeText(this, "Clicked!", Toast.LENGTH_SHORT).show()
}
```

---

# 🔟 View Binding (Modern Approach)

Recommended alternative to `findViewById`.

Enable in **build.gradle**:

```kotlin
buildFeatures {
    viewBinding = true
}
```

Use in Activity:

```kotlin
private lateinit var binding: ActivityMainBinding
```

Inside `onCreate`:

```kotlin
binding = ActivityMainBinding.inflate(layoutInflater)
setContentView(binding.root)

binding.button.setOnClickListener {
    Toast.makeText(this, "Clicked!", Toast.LENGTH_SHORT).show()
}
```

Advantages:

✔ Safer
✔ No null errors
✔ Cleaner code

# 1️⃣1️⃣ RecyclerView (Important for Lists)

RecyclerView is used to **display lists of data**.

Example:

• Chat messages
• Products
• Notifications

RecyclerView works with two main components.

---

## Adapter

Connects **data → UI**.

Example:

```
List of users → RecyclerView → Screen
```

The adapter **creates and binds views**.

---

## ViewHolder

Stores references to the UI elements for **better performance**.

Example structure:

```
RecyclerView
   │
   ├── Adapter
   │       │
   │       └── ViewHolder
```

# 1️⃣2️⃣ XML vs Jetpack Compose (Exam Cheat Sheet)

| XML              | Compose                  |
| ---------------- | ------------------------ |
| LinearLayout     | Column / Row             |
| ConstraintLayout | ConstraintLayout Compose |
| RecyclerView     | LazyColumn               |
| TextView         | Text()                   |
| Button           | Button()                 |
| findViewById     | Not needed               |

Compose is **state-driven**, while XML uses **view references**.

---

# 1️⃣3️⃣ XML Best Practices

✔ Use **ConstraintLayout** for complex screens
✔ Avoid **deep nested layouts**
✔ Use **dp** for sizes
✔ Use **sp** for text
✔ Use **strings.xml instead of hardcoded text**

Example:

```xml
android:text="@string/login"
```
